# Contributing to Laravel TRA VFD Integration

First off, thank you for considering contributing to this package! It's people like you that make the Tanzanian developer ecosystem great.

## Code of Conduct
Please be respectful and professional in all interactions.

## How Can I Contribute?

### Reporting Bugs
- Use the GitHub Issue Tracker.
- Describe the bug and include steps to reproduce it.
- Include your Laravel version and PHP version.

### Suggesting Enhancements
- Open an issue to discuss the enhancement before starting work.
- Focus on features that benefit the wider TRA VFD community (e.g., support for Z-Reports or Adjustments).

### Pull Requests
1. **Fork the repository** and create your branch from `main`.
2. **Include Authorship Headers**: All new files must include the copyright header for John M Kagaruki.
3. **Write Tests**: If you add a feature, please include a test case.
4. **Follow PSR-12**: Ensure your code follows standard PHP styling.

## Security Vulnerabilities
If you discover a security vulnerability (especially regarding XML signing or PFX handling), please email **john@eyecuemedia.co.tz** directly instead of using the public issue tracker.
